# 제품과 불변 조건

## 제품 약속

DamSet은 iOS 26+ iPhone에서 루틴 선택부터 실제 세트 기록, 휴식, 잠금 화면
확인, 운동 이력까지 흐름을 끊지 않는 운동 파트너다. 운동 중 앱을 오래 보게
하는 정교함보다 짧은 동작, 큰 숫자, 한 손 조작, 잠긴 상태의 진행 가시성이
우선이다.

## 핵심 사용자 흐름

```text
루틴 선택
→ 오늘 수행할 운동 선택
→ 실제 횟수/시간·무게 조정
→ 세트 완료
→ 벽시계 기준 휴식과 종료 알림
→ 다음 세트
→ 완료한 세트만 이력 저장
```

운동 시작 전에 고른 종목과 세트 수는 세션에만 적용되며 재사용하는 원본
루틴을 바꾸지 않는다.

루틴 편집에서는 공통 목표를 기본값으로 두되, 필요하면 각 세트의 무게와
횟수를 독립적으로 덮어쓸 수 있다. 예를 들어 워밍업과 작업 세트를 한 운동
안에 `10kg × 10회`, `15kg × 8회`, `15kg × 5회`로 계획한다.

## UX 불변 조건

- 현재 운동, 세트 위치, 목표, 실제 수행량, 다음 행동은 일반 iPhone 높이에서
  빠르게 보여야 한다.
- 주 행동의 탭 영역은 최소 44×44pt다. 운동 중 반복 조절은 더 큰 원형
  컨트롤을 사용한다.
- 반복 기반과 시간 기반 운동은 같은 세션 흐름을 공유하지만 서로 다른 진행
  단위를 정확히 표시하고 저장한다.
- 휴식 남은 시간은 틱 누적값이 아니라 실제 `resumeAt` 시각에서 계산한다.
- 잠금 화면의 조절은 앱과 같은 정규 세션을 바꾸며 오래된 Live Activity가
  새 세션을 건드리면 안 된다.
- 잠금 화면에서 수행량을 빠르게 조절할 때 위젯 전체가 번쩍이지 않아야 하며,
  숫자만 증감 방향을 드러내며 전환돼야 한다.
- 무게 조절은 앱과 잠금 화면 모두 1kg 단위로 일관되게 적용한다.
- 휴식 종료 뒤에는 잠긴 상태에서도 다음 세트 카드가 제때 나타나야 하며,
  이를 위해 별도 서버를 요구하지 않는다.
- 저장 실패가 난 뒤 완료 화면으로 낙관적으로 넘어가 완료 세트를 잃어서는
  안 된다.
- 포그라운드 휴식 종료음은 다른 음악을 중지하지 않고 잠시 duck한 뒤
  복구하는 것이 이상적이다. 잠금/백그라운드는 iOS 정책상 로컬 알림과
  시스템 사운드가 허용된 폴백이다.
- 완료 이력에는 실제로 완료한 세트만 들어가며 사용자는 이후 값을 수정하거나
  기록을 삭제할 수 있다.

## 현재 범위

포함:

- 사용자 루틴과 운동별 무게·횟수/시간·휴식 설정
- 운동별 공통 목표와 세트별 무게·횟수 덮어쓰기
- 세션 한정 종목 선택과 세트 추가
- 실제 수행량 수정, 휴식 건너뛰기/조절, 완료 취소
- Live Activity와 Dynamic Island
- 로컬 알림, 포그라운드 오디오·햅틱
- 로컬 JSON 저장, 달력형 이력, 볼륨·진행 분석

현재 제외:

- Apple Watch, iPad, macOS 정식 제품
- 계정·클라우드 동기화·소셜 기능
- AI 코칭

SwiftPM의 macOS 앱 셸은 UI 자동화용 폴백이며 제품 플랫폼 확장을 뜻하지
않는다.

## 디자인 방향

네이티브 SwiftUI, 시스템 타이포그래피, SF Symbols를 우선한다. 현재 제품
표면은 분체 도장 랙과 새틴 바벨에서 가져온 탄소색 배경, 모따기 금속 패널,
강철 구분선, 장비 적색 CTA를 사용한다. 큰 숫자와 얇은 적색 발광은 운동
상태와 다음 행동에만 집중한다.

Live Activity는 운동명·세트·수행량·휴식 상태처럼 잠깐 보고 판단할 정보만
담는다. README나 마케팅 이미지도 임의의 피트니스 대시보드를 만들지 말고
실기기 캡처를 화면 근거로 사용하며, 캡처용 미러링 배지와 포인터는 제거한다.
자세한 시각 체크리스트는 `docs/design-notes.md`, 구현 색과 재질은
`DamSetApp/DamSetDesign.swift`가 근거다.
